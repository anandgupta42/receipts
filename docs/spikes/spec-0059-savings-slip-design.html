The savings slip

One 9-line block, set entirely in the receipt's own idiom: the savings headline is typeset like the receipt's TOTAL line, and every waste line carries its fix directly underneath as a one-line → rule. No second fence, no prose paragraphs. All lines are exactly 50 columns, verified.

Anatomy

①Headline — caps + dotted leaders + right-aligned amount, the receipt's TOTAL voice. The ≤ makes it a ceiling. Same string becomes the collapsed summary row, so the answer shows before any click.extracted
②Hedge — the receipt's existing "arithmetic, not a prediction" idiom, plus the share of total. One muted line, I3-honest.extracted
③Evidence line — waste class, run length, wall clock, dollars, cost-descending — prefixed with the main receipt's own waste glyph (⚠ loops, ≈ repriced spans) so slip and receipt visibly rhyme.extracted
④Rule line — the fix, one line, arrow-prefixed, keyed to the waste class. Fixed strings compressed from SPEC-0013's templates; banned-phrase tested; never model-generated.static template
⑤Covers footer — what the slip accounts for (SPEC-0042's honesty line, compacted to one line).extracted

The slip is a renderer, not a PR feature: --handoff gains it between its header and covers: line, and the PR section embeds the identical output. One code path — the two surfaces cannot drift (the same discipline handoff.ts already applies to waste-line wording).

Where the slip follows other content — here, the handoff header — it opens with the receipt's 50-dash rule, the same seam the receipt draws before TOTAL. Standalone in the PR fence, the headline opens directly.

Pasted into an agent's next prompt, the slip reads as instructions with receipts attached: each → rule sits directly under the dollars that justify it.

Rule-line strings (the full static set)

One per waste class, fixed in-spec, ≤ 48 characters so they never wrap:

stuck-loop → change or stop after two identical failures
trivial-spans → route short replies to a cheaper model
context-thrash → clear or split context at task boundaries

The context-thrash string is SPEC-0017's existing suggestion verbatim; the other two are one-line compressions of SPEC-0013's templates (the long forms stay in the standing-rule section, which remains behind the 3-session recurrence gate). Spec checklist: shared renderer in src/receipt/handoff.ts; banned-phrase test coverage for the new strings; section omitted when no waste fired and under --no-details; SPEC-0027 HTML-artifact parity; SPEC-0043 telemetry event; spec id claimed against main + open PRs.
