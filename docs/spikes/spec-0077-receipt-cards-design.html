Shareable receipt cards

A --card output flag — folded into the commands you already run (pr and the session default), sitting next to --svg and --png. It re-renders the receipt as a 1200×630 social card and hands you a one-gesture way to post it. v1 ships the unit card (one PR or session); the recap for a window of time is SPEC-0078.

  • flag --card
  • scopes session · pr · recap → SPEC-0078
  • export PNG + SVG · 1200×630
  • hosted nothing — image travels on its own

Unit card

aireceipts pr --card

One PR or one session. The itemization is the point — same dot-leader breakdown as the terminal receipt, tuned to read at a glance in a feed. The card is the hook: it links out to the full receipt — the sticky PR comment, with every session and expandable detail. The image summarizes; the link carries the depth.
Note: the session card additionally carries the cheaper-model line (↘ same tokens on haiku…); the PR card omits it — an aggregate repricing can't be made price-safe, so it stays session-only.

The share step

one gesture, nothing hosted

The card lands on disk and on your clipboard, with the caption pre-written and the intent URLs printed for you to click. Web composers can't attach an image for you — so "easy" is: one paste, or one drag. No account, no backend, no upload. At PR level the caption also carries a link to the full receipt; the repo/PR never gets burned into the image (opt in with --link).

$ aireceipts pr 189 --card ✓ Card saved ~/Downloads/aireceipts-pr-189.png (1200×630) ✓ Caption + image + link copied to clipboard caption "PR #189 — $3.13 across 3 sessions. Full receipt ↓" link github.com/…/pull/189#issuecomment-… ← full receipt, every session Post it: X / LinkedIn · caption + link prefilled ↳ drag the image in — composers can't attach it for you

Aggregate scopes (week, --since, in SPEC-0078) have no repo to anchor a URL, so they're image-only — the card is the whole shareable unit and the full breakdown stays local in your terminal.

What rides on the card

aggregate-only by default

Same rule as --no-details: the shareable card carries figures, never the story of what you worked on.

On the card

  • ✓ Cost, in dollars
  • ✓ Token counts & cache-hit rate
  • ✓ Model mix & tool breakdown
  • ✓ Session / PR counts, active days, dates
  • ✓ Cheaper-model arithmetic & fun equivalence

Never (by default)

  • ✗ Prompts, replies, or transcript
  • ✗ Source code & file contents
  • ✗ Session titles (prompt-derived)
  • ✗ Repo, branch & project names
  • ✗ File paths

The card is always sanitized: --include-projects / --include-titles are refused with --card.
