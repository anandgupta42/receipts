## QA edge findings (lead-run, this session)

- Empty or whitespace-only `AIRECEIPTS_RECEIPT_EXEMPT_GLOBS` disables exemptions
  entirely (`""` is not nullish, so the `release/*` default does not apply;
  `isExemptRef` returns false for empty/whitespace glob lists). Sensible semantics
  ("no exemptions"), verified by direct execution. Not separately documented —
  accepted, noted for docs polish (#249 bucket).
- `AIRECEIPTS_REQUIRE_PR_RECEIPT` requires the literal `true` (uppercase `TRUE`
  stays off) — unchanged from v0.10.0 and documented as "exactly true".
- 10k-char unicode ref against `release/*`: correct verdict in <1 ms.
- Verdict matrix unchanged for: enforcement off, forks, found-receipt paths
  (pinned by `test/cli/pr-check.test.ts`, 18 tests).

## Changelog draft (user language)

- `pr-check` enforcement now skips release branches by default: branches matching
  `release/*` stay notice-only when `AIRECEIPTS_REQUIRE_PR_RECEIPT=true`, because
  release checkouts have no agent session to attest. Override with the
  `AIRECEIPTS_RECEIPT_EXEMPT_GLOBS` repo variable or the `--exempt-globs` flag
  (space-separated anchored globs, `*` wildcards only; set it empty to enforce
  everywhere).
- The adopter workflow snippet (`docs/adopt/pr-check-caller.yml` and
  `aireceipts integrations`) forwards the new variable; existing callers keep
  working unchanged.
- Reliability hardening in the dogfood pipeline (#259): relative `core.hooksPath`
  self-heal and exempt-glob support in the reusable check workflow — the outage
  class that silently dropped PR receipts for a week can no longer stay silent.

## Open risks
