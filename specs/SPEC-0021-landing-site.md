# SPEC-0021 · Landing site

Invariants: I6 (facts, never rankings); the honesty rules quoted on the page must be
byte-true to the product's behavior. The site makes no external requests (R5) — a
"local, no cloud" product whose homepage phones nowhere.

## Purpose

One hand-crafted page that converts a skeptical developer in 30 seconds: what it
costs to run their agent, that this tool shows it in one command, and that every
number is constructed to be believable. Hosted on GitHub Pages (public repos, free,
zero ops), live at the OSS flip; aireceipts.io CNAMEs to it.

## Design (lead-authored, binding — copy is verbatim, not a suggestion)

**The signature: the entire page is one long receipt.** Paper background `#f4ede0`
with a faint grain, a single 640px receipt column, scalloped top and bottom edges,
dashed-rule section bands, system monospace throughout. Accent `#3947C2`; the flag
red `#B3372E` appears exactly once (the waste row). No framework, no build step, one
HTML file, no JS except the copy-button (and it degrades to plain text).

**Band 1 — masthead.**
`AIRECEIPTS` wordmark (letterspaced caps) ·
H1: **"Your AI coding agent just worked for 33 minutes. Here's the receipt."**
Sub: "One command. No account, no API key, no upload — it reads the transcripts
your agent already writes to disk."
Copy-box: `npx aireceipts` (the single CTA; a quiet second link: "or read the
methodology first").

**Band 2 — the hero receipt.** The real dark-theme receipt PNG (a genuine session,
title visible, waste line firing, TOTAL + same-tokens compare rows adjacent).
Caption: "A real session. Every number priced from cited, dated vendor tables."

**Band 3 — `--- WHY YOU'D WANT THIS ---`** — value props as receipt line-items with
dotted leaders (label left, payoff right):

```
Know the cost before the invoice does ......... every session, priced
Catch the waste ................... stuck loops & tiny turns, in $
Fix it permanently ......... paste-ready CLAUDE.md standing rules
Put receipts on your PRs ....... what the change cost to produce
See the week ............ totals, per-agent split, honest deltas
```

Under it, one paragraph of prose (the "why" story): *"Metered AI coding is the new
cloud bill: invisible while you work, shocking at month-end. aireceipts turns the
transcripts already sitting on your disk into itemized receipts — so you learn which
habits burn money (re-running a failing command five times: $0.08 and 3m45s), which
model your trivial turns actually need, and what each PR cost to produce. Then it
helps you fix it: every receipt ends with a paste-ready handoff that tells your
agent what to do cheaper next time."*

**Band 4 — `--- WHY YOU CAN BELIEVE IT ---`** — the four honesty rules as
line-items (these sentences are public promises; byte-match the README):
no fabricated dollars, ever · every price cited with URL + date + excerpt ·
comparisons are arithmetic, not predictions · deterministic: same transcript in,
byte-identical receipt out. Plus the telemetry line: "Diagnostics-only telemetry,
fully disclosed, one env var to kill — and your transcripts never leave your
machine."

**Band 5 — `--- ALSO ON THE MENU ---`** — three small text-rendered receipts
side-by-side (compare, week digest, statusline one-liner), each with a one-line
caption. Real renders, never mockups.

**Band 6 — footer.** `CARDHOLDER: you` · `THANK YOU FOR YOUR TOKENS` ·
deterministic pipe-barcode · GitHub / docs / Apache-2.0 · "buy me a samosa 🥟".
OG card = the dark receipt PNG; title "aireceipts — receipts for your AI coding
agent".

**Tone rules:** no adjectives the product can't prove ("blazing", "beautiful"
banned); numbers over adverbs; the joke stays deadpan (the page never winks at its
own receipt conceit).

## Requirements

- **R1** `site/index.html` — single self-contained file (inline CSS, assets as
  files beside it), matching the Design section exactly; responsive to 360px;
  reduced-motion respected; passes basic a11y (landmarks, contrast, alt text).
- **R2** Pages deploy workflow (`actions/deploy-pages` from `site/`), enabled at
  flip; CNAME file for aireceipts.io.
- **R3** Zero external requests (fonts = system stack; analytics = none — CI check
  greps for `http` in the HTML and allowlists only the GitHub links).
- **R4** All receipt imagery generated by the product from fixtures via a script
  (`scripts/site-assets.mts`), regenerated + byte-checked in CI like goldens — the
  site can never drift from real output.
- **R5** Lighthouse ≥95 performance/a11y in CI (or documented waiver per run).

## Non-goals

A docs site (VitePress or similar — its own M4 spec when docs outgrow repo browsing);
a blog; analytics of any kind; framework/build tooling; dark-mode toggle (the hero
PNG is dark; the paper page is the light); pricing/SaaS pages.
